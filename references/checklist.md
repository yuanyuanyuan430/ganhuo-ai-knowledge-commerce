# Knowledge Commerce Pipeline Checklist

Score each area as `0 missing`, `1 partial`, or `2 ready`. Prioritize fixes that improve source quality, publishing speed, trust, and conversion.

| Area | Ready signal | Common fix | Verify |
| --- | --- | --- | --- |
| Offer | Niche, user pain, deliverable, price, and proof are clear | Rewrite the offer as one concrete transformation | Ask a target user to explain what they get |
| Source cards | Every source has URL/contract, owner, permission, usable scope, and risk note | Build a Feishu Bitable or Markdown source-card folder | Randomly audit 5 sources |
| Original angle | Each article adds tests, experience, examples, templates, or judgment | Add a human story bank and test log | Highlight original sections in each draft |
| Knowledge base | Topics, FAQs, templates, cases, objections, and glossary are organized | Create topic clusters and retrieval tags | Ask 10 user questions and inspect retrieved context |
| Voice | Tone, sentence rhythm, examples, CTA, and taboo claims are defined | Write a 1-page voice guide | Compare 3 drafts for consistency |
| Content factory | Topic -> outline -> draft -> review -> platform variant is repeatable | Use one prompt pack and one review checklist | Produce 3 drafts with the same workflow |
| Review | Fact, source, privacy, claim, and platform checks happen before publishing | Add two review states: topic approval and final approval | Record reviewer and status |
| Distribution | 公众号, 知识星球, 朋友圈, 企微, and low-ticket product roles are separated | Create a platform action table | Track each content ID across channels |
| Metrics | Reads, completion, follows, comments, paid conversion, refund, and complaints are tracked | Add a weekly dashboard | Run one weekly review |
